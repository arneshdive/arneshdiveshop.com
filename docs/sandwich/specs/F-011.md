# F-011: Multi-step checkout flow (guest checkout, auto-registration, confirmation)

> **Module:** Checkout · **Priority:** 7/100 · **Depends on:** F-002, F-005, F-010 · **Source:** prd.md:69-75

Contact → Shipping → Payment → Confirm checkout with Indonesia-specific address fields, guest checkout without prior registration, auto-registration of a customer account on order completion, order summary review, and confirmation page.

## Scope

**In:**
- Multi-step checkout: Contact → Shipping → Payment → Confirm
- Indonesia-specific shipping address fields (province, city, postal code)
- Guest checkout without prior registration
- Auto-registration of a customer account on order completion using collected contact/shipping data, with duplicate-email lookup/linking
- Order summary review before payment
- Order confirmation page with order details

**Out:**
- Payment execution itself (separate Midtrans feature)
- Saved-address reuse for guests

## Acceptance Criteria

- [ ] **AC1** — A guest can complete checkout without registering beforehand
- [ ] **AC2** — Shipping form validates Indonesia-specific fields (province, city, postal code)
- [ ] **AC3** — On order completion, a customer account is auto-created from the checkout contact data
- [ ] **AC4** — If the guest's email already has an account, the order links to that existing account instead of creating a duplicate
- [ ] **AC5** — Customer sees an order confirmation page with order details after checkout
