# F-002: Core authentication (registration, login, session, RBAC, rate limiting)

> **Module:** Authentication · **Priority:** 10/100 · **Depends on:** F-001 · **Source:** prd.md:29-34

Custom JWT session auth via jose: email/password registration with validation, secure login with HTTP-only cookie sessions, role-based access control (customer/admin/super_admin), and rate limiting on auth endpoints.

## Scope

**In:**
- Email/password registration with validation (name, email, password, phone)
- Login with credential verification against stored password hash (lib/auth/password.ts)
- HTTP-only cookie session using jose JWT, containing user ID and role
- Role-based access control: customer, admin, super_admin
- Rate limiting on auth endpoints (register, login)

**Out:**
- Email verification (separate feature)
- Forgot password flow (separate feature)
- OAuth/social login (not in brief)

## Acceptance Criteria

- [ ] **AC1** — A customer can register with email/password and receive a session
- [ ] **AC2** — A customer can log in and receive an HTTP-only JWT session cookie
- [ ] **AC3** — Admin-only routes reject requests from a customer-role session
- [ ] **AC4** — Repeated failed login attempts are rate-limited
