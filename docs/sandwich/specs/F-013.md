# F-013: Order creation from cart (atomic transaction)

> **Module:** Order Management · **Priority:** 14/100 · **Depends on:** F-005, F-011, F-012 · **Source:** prd.md:98

Order + orderItems + payment record created atomically in a single DB transaction on checkout completion, with an idempotency key to prevent duplicate orders from double form-submits.

## Scope

**In:**
- Order creation from cart contents in a single DB transaction (order + orderItems + payment record)
- Idempotency key to prevent duplicate orders from double form-submits
- Order number format ORD-YYYYMMDD-XXXX
- Initial order status: pending_payment

**Out:**
- Order editing after creation (handled by status-update feature)

## Acceptance Criteria

- [ ] **AC1** — Completing checkout creates exactly one order with its items and payment record, even under a double-submit
- [ ] **AC2** — Order numbers follow the ORD-YYYYMMDD-XXXX format
- [ ] **AC3** — A failed transaction leaves no partial order/items/payment rows behind
