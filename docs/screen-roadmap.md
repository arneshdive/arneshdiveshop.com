# E-Commerce Screen Roadmap

A prioritized list of screens for building a complete online shop experience.

## Core Screens (Priority Order)

### 1. Homepage ✅
**Status:** Wireframe complete (`design.html`)

The storefront's front door. Key elements:
- Announcement bar
- Navigation
- Hero section
- Category navigation
- Featured products
- Promotional banners
- Social proof / UGC
- Value propositions
- Footer

---

### 2. Product Listing Page (PLP)
**Purpose:** Browse products within a category or search results

Key elements:
- Breadcrumb navigation
- Category title & description
- Filter sidebar (price, size, color, etc.)
- Sort options
- Product grid
- Pagination or infinite scroll
- Quick view option

**Real-world examples:**
- ASOS category pages
- Zara collection pages
- Nike product listings

---

### 3. Product Detail Page (PDP)
**Purpose:** View full product info and add to cart

Key elements:
- Product image gallery (with zoom)
- Product title, price, description
- Size selector
- Color/variant selector
- Add to cart button
- Size guide
- Product details/accords
- Reviews section
- Related products
- Recently viewed

**Real-world examples:**
- Apple product pages
- Glossier product pages
- Everlane product pages

---

### 4. Cart Page
**Purpose:** Review items before checkout

Key elements:
- Cart item list with images
- Quantity adjusters
- Remove item option
- Price summary (subtotal, shipping, tax)
- Promo code input
- Continue shopping link
- Proceed to checkout button

**Real-world examples:**
- Stripe checkout cart
- Shopify default cart
- Amazon cart

---

### 5. Checkout Flow
**Purpose:** Complete purchase

Key stages:
1. **Contact info** — Email, phone
2. **Shipping** — Address, method selection
3. **Payment** — Card details, billing address
4. **Review** — Final confirmation

Key elements:
- Progress indicator
- Form validation
- Guest checkout option
- Saved addresses (if logged in)
- Order summary sidebar
- Trust badges

**Real-world examples:**
- Stripe checkout
- Shopify checkout
- Apple checkout

---

### 6. Search Results
**Purpose:** Find specific products

Key elements:
- Search query display
- Result count
- Filters
- Sort options
- Product grid
- "Did you mean?" suggestions
- No results state

---

## Secondary Screens

- **Account Dashboard** — Orders, saved items, settings
- **Login / Register** — Auth flows
- **Wishlist / Saved Items**
- **Order Confirmation** — Post-purchase
- **Store Locator** — Physical stores
- **Contact Page**
- **FAQ / Help Center**

---

## Notes

- Each screen should be designed as a wireframe first
- Wireframes should be based on proven real-world patterns
- Mobile responsiveness is required for all screens
- Consider accessibility (ARIA, keyboard navigation, contrast)
