# F-012: Midtrans payment integration (Snap, VA, e-wallets, QRIS, webhook, idempotency)

> **Module:** Payment · **Priority:** 6/100 · **Depends on:** F-005, F-011 · **Source:** prd.md:82-90

Midtrans Snap checkout supporting cards, bank transfer (VA), e-wallets (GoPay/OVO/ShopeePay), and QRIS, behind a PaymentProvider interface; signature-verified webhook handler; payment status tracking; idempotency keys to prevent duplicate charges.

## Scope

**In:**
- Midtrans Snap integration for checkout
- Support for credit/debit cards, bank transfer (VA), e-wallets (GoPay, OVO, ShopeePay), QRIS
- PaymentProvider interface abstracting Midtrans (swappable for Stripe/Xendit later)
- Webhook handler with signature verification before updating order status
- Payment status tracking (pending, paid, failed, expired)
- Idempotency keys to prevent duplicate charges

**Out:**
- Any payment provider other than Midtrans
- Refund/partial-refund flows (not stated)

## Acceptance Criteria

- [ ] **AC1** — Customer can pay via Midtrans Snap using at least one card, VA, e-wallet, and QRIS method
- [ ] **AC2** — Webhook handler rejects requests with an invalid Midtrans signature
- [ ] **AC3** — A verified webhook updates the order's payment status correctly
- [ ] **AC4** — Submitting the same order twice does not create duplicate charges
