# F-003: Email verification (OTP) for new accounts

> **Module:** Authentication · **Priority:** 8/100 · **Depends on:** F-002 · **Source:** prd.md:32

New accounts start unverified; system sends an OTP via Mailgun that the customer must confirm before full access.

## Scope

**In:**
- Generate OTP on registration and send verification email via Mailgun
- Verify OTP and mark account status as verified
- OTP expiry handling

**Out:**
- SMS-based verification

## Acceptance Criteria

- [ ] **AC1** — A newly registered account starts in 'unverified' status
- [ ] **AC2** — Customer receives a verification email with OTP/link after registering
- [ ] **AC3** — Clicking/submitting a valid, unexpired OTP marks the account verified
