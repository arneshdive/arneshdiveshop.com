# F-015: Order status updates & audit trail

> **Module:** Order Management · **Priority:** 14/100 · **Depends on:** F-014 · **Source:** prd.md:101-102

Admin updates order status through the fulfillment workflow (processing, shipped, delivered, cancelled) with an audit trail of changes.

## Scope

**In:**
- Order status transitions: pending_payment → processing → shipped → delivered (plus cancelled)
- Audit trail recording each status change

**Out:**
- Automated status transitions triggered by carrier tracking webhooks (not stated)

## Acceptance Criteria

- [ ] **AC1** — Admin can move an order through processing → shipped → delivered
- [ ] **AC2** — Every status change is recorded in an audit trail with who/when
