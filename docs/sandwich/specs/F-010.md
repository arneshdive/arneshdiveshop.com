# F-010: Server-side shopping cart (persistence, add/remove/update, real-time totals)

> **Module:** Shopping Cart · **Priority:** 11/100 · **Depends on:** F-001, F-002 · **Source:** prd.md:58-62

Cart persisted to the database for logged-in users, add/remove/update items with quantity validation, real-time total calculation, synced with the existing Zustand client store.

## Scope

**In:**
- Server-side cart storage for logged-in users
- Cart persistence across sessions
- Add/remove/update cart items with quantity validation
- Real-time cart total calculation

**Out:**
- Guest cart persistence beyond the client-side Zustand store

## Acceptance Criteria

- [ ] **AC1** — A logged-in customer's cart persists across sessions
- [ ] **AC2** — Adding, removing, and updating quantity all reflect correctly in the cart total
- [ ] **AC3** — Quantity cannot be set below 1 or above a sane validated max
