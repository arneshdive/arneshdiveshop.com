# F-004: Forgot password / reset flow (OTP)

> **Module:** Authentication · **Priority:** 8/100 · **Depends on:** F-002 · **Source:** prd.md:31

Customer requests a password reset, receives an OTP via Mailgun, and sets a new password after OTP verification.

## Scope

**In:**
- Request password reset by email, generate OTP, send via Mailgun
- Verify OTP and allow setting a new password
- Invalidate OTP after use or expiry

**Out:**
- SMS-based reset

## Acceptance Criteria

- [ ] **AC1** — Customer can request a reset OTP for a registered email
- [ ] **AC2** — A valid, unexpired OTP allows setting a new password
- [ ] **AC3** — An expired or already-used OTP is rejected
